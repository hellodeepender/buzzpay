# Accessibility checklist

Use this checklist when reviewing BuzzPay calculator and recommendation pages:

- Keyboard can reach all calculator inputs, form controls, report actions, AI actions, sidebar links, and recommendation cards.
- Every visible input has a matching label, and required fields are marked clearly.
- Form errors appear in text and are announced to assistive technology.
- Loading, success, and error updates use a live region.
- Focus states are visible on links, buttons, and inputs.
- Tap targets are large enough on mobile and do not overlap.
- Color is not the only signal for error or success states.
- Report forms require consent before submission.
- The AI explanation button only runs on explicit click.
